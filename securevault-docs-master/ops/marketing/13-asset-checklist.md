## Asset Specifications

### Image Specifications

**Screenshots:**
- Format: PNG (with transparency)
- Dimensions: 1920x1080 or higher
- File size: <500KB (optimized)

**Social Media:**
- Format: PNG or JPG
- Dimensions: Platform-specific
- File size: <200KB (optimized)

**Logos:**
- Format: SVG (preferred) or PNG
- Dimensions: Scalable
- File size: <50KB

### Video Specifications

**Format:** MP4 (H.264)
**Dimensions:** 1920x1080 (16:9)
**Frame rate:** 30fps
**File size:** <10MB (optimized)
**Duration:** 60 seconds or less

### PDF Specifications

**Format:** PDF
**File size:** <2MB (optimized)
**Pages:** 1-2 pages
**Print-ready:** Yes

### Naming Convention

**Format:** `[type]-[name]-[version].[ext]`

**Examples:**
- `video-hero-demo-v1.mp4`
- `screenshot-portal-v1.png`
- `case-study-partner-1-v1.pdf`

## Asset Delivery

### Requirements

- All assets must be optimized for web
- Provide source files when applicable
- Include usage guidelines
- Maintain brand consistency

### Approval Process

1. **Draft** → Marketing review
2. **Revision** → PM accuracy check
3. **Final** → Brand approval
4. **Published** → Live on website

## Maintenance

### Updates

- Review assets quarterly
- Update screenshots when UI changes
- Refresh case studies annually
- Update pricing charts when plans change

### Version Control

- Maintain version numbers
- Archive old versions
- Document changes
- Keep master files
