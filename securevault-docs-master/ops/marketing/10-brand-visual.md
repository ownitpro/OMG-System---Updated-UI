# Brand & Visual

## Tone of Voice

**Style:** Clear, calm, direct

**Reading Level:** 9th-grade reading level

**Jargon:** Zero jargon - use simple, everyday language

**Examples:**
- ✅ "Upload files easily"
- ❌ "Leverage our file ingestion pipeline"

- ✅ "See where you are with usage meters"
- ❌ "Monitor your resource consumption metrics"

## CTA Style

### Primary CTA
- **Color:** High-contrast green button (#2bd576)
- **Style:** Solid, prominent
- **Text:** Action-oriented (Get started free, Try the demo)

### Secondary CTA
- **Color:** Outline style
- **Style:** Border, less prominent
- **Text:** Alternative actions (See pricing, Contact Sales)

### CTA Best Practices
- Use action verbs
- Be specific
- Create urgency (when appropriate)
- Test different variations

## Logos & Marks

### Primary Logo
- **SVD logo lock-up** from app header
- Use consistently across all materials
- Maintain aspect ratio
- Minimum size: 24px height

### Footer
- **"Powered by OMGsystems — © 2025"**
- Use on all marketing pages
- Consistent placement and styling

### Logo Usage
- Always use approved logo files
- Don't modify colors or proportions
- Maintain clear space around logo
- Use on light or dark backgrounds as appropriate

## Color Palette

### Primary Colors
- **Green (Primary):** #2bd576
- **Black/Dark:** #0b0f0c
- **White:** #ffffff

### Accent Colors
- **Emerald:** #10b981 (for highlights)
- **Zinc:** #71717a (for secondary text)

### Usage Guidelines
- Green for primary CTAs and highlights
- Black/Dark for backgrounds and primary text
- White for text on dark backgrounds
- Use colors consistently across all materials

## Typography

### Headings
- **Font:** System font stack (sans-serif)
- **Weight:** Bold (700) for H1, Semibold (600) for H2
- **Size:** Responsive (larger on desktop)

### Body Text
- **Font:** System font stack (sans-serif)
- **Weight:** Regular (400)
- **Size:** 16px base, responsive

### Code/Technical
- **Font:** Monospace font stack
- **Weight:** Regular (400)
- **Size:** 14px

## Imagery

### Screenshots
- Use actual product screenshots
- Maintain consistent styling
- Add subtle shadows/borders
- Keep file sizes optimized

### Icons
- Use consistent icon set (Lucide icons)
- Maintain size consistency
- Use appropriate colors
- Keep icons simple and clear

### Illustrations
- Use sparingly
- Maintain brand consistency
- Keep illustrations simple
- Use brand colors

## Layout Guidelines

### Spacing
- Consistent padding and margins
- Use grid system
- Maintain visual hierarchy
- Ensure readability

### Responsive Design
- Mobile-first approach
- Breakpoints: mobile, tablet, desktop
- Test on multiple devices
- Ensure touch targets are adequate

## Brand Consistency

### Do's
- ✅ Use approved logo files
- ✅ Follow color palette
- ✅ Maintain consistent tone
- ✅ Use clear, simple language
- ✅ Test on multiple devices

### Don'ts
- ❌ Modify logo colors or proportions
- ❌ Use jargon or technical terms
- ❌ Inconsistent spacing or styling
- ❌ Cluttered layouts
- ❌ Small text or touch targets

## Asset Guidelines

### File Formats
- **Logos:** SVG (preferred), PNG (with transparency)
- **Screenshots:** PNG, optimized
- **Icons:** SVG (preferred), PNG
- **Images:** JPG (photos), PNG (graphics)

### File Naming
- Use descriptive names
- Include dimensions if applicable
- Use lowercase with hyphens
- Example: `logo-svd-primary.svg`

### File Organization
- Organize by type (logos, screenshots, icons)
- Use version control
- Maintain master files
- Keep optimized versions for web
